# 架构说明

```
┌─────────────────────────────────────────────────────────┐
│                    微信小程序前端                         │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐      │
│  │  首页   │ │ 场景选择 │ │ 照片上传 │ │ 结果展示 │      │
│  └─────────┘ └─────────┘ └─────────┘ └─────────┘      │
└─────────────────────┬───────────────────────────────────┘
                      │ wx.cloud.callFunction

│                    微信云开发                             │
│  ┌─────────────────────────────────────────────────┐   │
│  │              generatePhoto 云函数                 │   │
│  │  1. 接收用户照片 fileID + 场景类型                │   │
│  │  2. 下载图片 → 转 base64                         │   │
│  │  3. 调用混元生图 API（异步）                      │   │
│  │  4. 创建任务记录到 cloud DB                       │   │
│  │  5. 轮询 API 直到生成完成                        │   │
│  │  6. 上传结果图到云存储                           │   │
│  │  7. 更新任务状态                                 │   │
│  └─────────────────────────────────────────────────┘   │
│                                                         │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐    │
│  │ 云数据库     │  │ 云存储      │  │ 定时触发器   │    │
│  │ (tasks 表)  │  │ (图片文件)  │  │ (清理过期)  │    │
│  └─────────────┘  └─────────────┘  └─────────────┘    │
└─────────────────────┬───────────────────────────────────┘
                      │ HTTPS API
┌─────────────────────▼───────────────────────────────────┐
│              腾讯混元文生图 API                           │
│  - 输入: 用户照片 (base64) + prompt (场景相关)           │
│  - 输出: 生成的职业形象照                                │
│  - 异步: 提交任务 → 轮询结果                            │
└─────────────────────────────────────────────────────────┘
```

## 数据模型

### tasks 表（云数据库）

| 字段 | 类型 | 说明 |
|------|------|------|
| taskId | string | 任务唯一 ID |
| openid | string | 用户 OpenID |
| fileID | string | 原图云存储 fileID |
| scene | string | 场景类型 (business/linkedin/creative) |
| status | string | pending/processing/completed/failed |
| resultImage | string | 结果图云存储 fileID |
| createdAt | date | 创建时间 |
| updatedAt | date | 更新时间 |

## 异步任务流程

1. 用户上传照片 → 云存储
2. 前端调用 generatePhoto 云函数
3. 云函数创建任务记录（status: pending）
4. 云函数调用混元 API 提交生图任务
5. 云函数轮询混元 API 等待结果（10-60s）
6. 生成完成后上传结果图到云存储
7. 更新任务状态为 completed
8. 前端轮询云函数查询任务状态
9. 完成后展示结果

## 变现架构

### 激励视频广告（个人主体可用）
- 广告位申请: mp.weixin.qq.com → 流量主 → 广告管理
- 门槛: 累计 UV ≥ 1000
- 接入: 微信小程序 Ad 组件
- 流程: 用户点击"看广告免费" → 播放激励视频 → 完成后解锁

### 月度会员（需企业主体）
- 接入: 微信虚拟支付 API
- 门槛: 企业主体 + 虚拟支付资质
- 流程: 用户购买月度会员 → 获得无限生成权限

## 安全考虑

- 用户照片仅用于生成，不存储原始图片超过 24 小时
- 云函数权限控制，仅允许已登录用户调用
- 结果图加水印，付费后移除
- 混元 API Key 存储在云函数环境变量中，不暴露给前端
